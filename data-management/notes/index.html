    Database Design

    1. Conceptual Data Model: must reflect the actual and possible states of the outside world. For example, if people can have two phone numbers, this must be possible.
    2. ER-Model
    3. Unified Modelling Language
    4. Schema/Logical Data Model: actual implementation of the conceptual data model in the chosen database technology.
    5. Normalization: According to wikipeda, "each elementary 'fact' is recorded in one place so that insertions, updates, and deletions automatically maintain consistency".
    6. Physical database design: affects performance, scalability, recovery, security, etc.
    7. Security
